## Story

**As a** TaskFlow frontend developer,
**I want** an API client and React hooks (useFlows, useCards, useTasks) that connect to the real backend,
**so that** frontend components can fetch and mutate TaskFlow data.

---

## Acceptance Criteria

1. [x] API client created at `packages/frontend/src/services/taskflow.api.ts` with typed methods for all CRUD operations
2. [x] Frontend types mirrored at `packages/frontend/src/types/taskflow.ts` (Flow, FlowColumn, Card, Task, DTOs)
3. [x] `useFlows` hook: list, create, update, delete flows with loading/error state
4. [x] `useCards(flowId)` hook: list cards for flow, create, update, delete, moveCard with loading/error state
5. [x] `useTasks(cardId)` hook: list tasks for card, create, update, delete with loading/error state
6. [x] API client uses `credentials: 'include'` for cookie-based auth
7. [x] Error handling: API errors propagated to hook error state
8. [x] All hooks return `refetch` function for manual refresh
