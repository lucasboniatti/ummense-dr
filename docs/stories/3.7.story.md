## Story

As a **platform operator**, I want to **implement rate limiting and concurrency controls to prevent resource exhaustion**, so that **one misbehaving automation doesn't destabilize the entire platform**.

---

## Acceptance Criteria

- [ ] **[CRITICAL]** Per-connector rate limiting (configurable: requests/sec, concurrent executions)
  - Admin panel: Set rate limits per connector (Slack, Zapier, REST APIs, etc.)
  - Settings: Requests per second (RPS) and max concurrent executions
  - Default: 10 RPS, 20 concurrent executions
  - Applied at webhook delivery layer (Story 3.2)
  - Exceeded limits: Requests queued and processed in order (FIFO)
- [ ] **[CRITICAL]** Global concurrency limit (configurable max parallel executions)
  - Admin setting: Max parallel automations (default 1000)
  - When limit exceeded: New executions queued, processed as others complete
  - Fair scheduling: FIFO queue prevents starvation
  - Queue depth visible in dashboard (Story 3.6)
- [ ] **[HIGH]** Queue management: Pending automations queued and processed in FIFO order with backpressure
  - Queue implementation: Bull, RabbitMQ, or Postgres-backed queue
  - Max queue size: Configurable (default 10K items)
  - Backpressure handling: Reject with 429 if queue full
  - Queue depth monitoring: API endpoint returns current depth, age of oldest item
- [ ] **[HIGH]** Circuit breaker pattern: Temporarily disable connector after N consecutive failures
  - Failure tracking: Count consecutive failures per connector
  - Threshold: N consecutive failures (configurable, default 5)
  - Activation: Connector marked as "degraded" or "offline"
  - Recovery: Exponential backoff to retry (1min, 5min, 15min, 1hr)
  - Manual override: Admin can force reset (clear failure count)
- [ ] **[HIGH]** Manual intervention: Pause/resume automations, clear execution queues
  - Dashboard controls (Story 3.6):
    - Pause button: Stop processing new executions (queued items remain)
    - Resume button: Resume processing from queue
    - Clear queue button: Delete all pending executions (with confirmation)
  - Per-connector pause: Pause specific connector without affecting others
  - Effect: Paused automations don't execute, webhooks don't deliver
- [ ] **[MEDIUM]** Metrics: Queue depth, active executions, connector availability
  - Real-time metrics (via WebSocket):
    - Current queue depth (global and per-connector)
    - Active executions (global count)
    - Connector status (healthy, degraded, offline)
    - Backpressure events (how many times 429 returned)
  - Historical metrics: Queue depth trend over last 7 days
  - Alerts: Queue depth >1000 or latency >5s triggers alert
- [ ] **[MEDIUM]** Graceful degradation: Failed integrations don't block other automations
  - Circuit breaker prevents cascading failures
  - One connector's failure doesn't affect other automations
  - Error isolation: Failures logged and monitored separately
  - Dashboard shows connector health status per integration

## Technical Notes

### Rate Limiting Implementation

```typescript
import { RateLimiterRes, RateLimiterMemory } from 'rate-limiter-flexible';

class ConnectorRateLimiter {
  private limiters: Map<string, RateLimiterMemory> = new Map();

  constructor(private defaultRPS: number = 10, private defaultConcurrent: number = 20) {}

  async checkLimit(connectorId: string): Promise<{ allowed: boolean; waitMs?: number }> {
    let limiter = this.limiters.get(connectorId);
    if (!limiter) {
      limiter = new RateLimiterMemory({
        points: this.defaultRPS,
        duration: 1
      });
      this.limiters.set(connectorId, limiter);
    }

    try {
      const res = await limiter.consume(1);  // Consume 1 point
      return { allowed: true };
    } catch (rateLimiterRes: any) {
      // Rate limit exceeded - return wait time
      return {
        allowed: false,
        waitMs: Math.ceil(rateLimiterRes.msBeforeNext)
      };
    }
  }

  async setLimit(connectorId: string, rps: number, concurrent: number) {
    // Update limiter configuration
    const newLimiter = new RateLimiterMemory({
      points: rps,
      duration: 1
    });
    this.limiters.set(connectorId, newLimiter);
    // Also update database for persistence
    await db('connector_rate_limits').update({
      rps,
      concurrent,
      updated_at: new Date()
    });
  }
}
```

### Execution Queue (Bull)

```typescript
import Queue from 'bull';

const executionQueue = new Queue('automations', {
  redis: { host: process.env.REDIS_HOST, port: 6379 },
  maxStalledCount: 2,
  stalledInterval: 5000
});

executionQueue.process(async (job) => {
  const { automationId, triggerData } = job.data;

  // Check global concurrency
  const activeCount = await getActiveExecutionCount();
  if (activeCount >= MAX_CONCURRENT) {
    // Re-queue for later
    throw new Error('Max concurrency reached');
  }

  // Check per-connector rate limit
  const connector = await getAutomationConnector(automationId);
  const rateLimitOk = await rateLimiter.checkLimit(connector.id);
  if (!rateLimitOk.allowed) {
    // Re-queue for later
    job.moveToDelayed(Date.now() + rateLimitOk.waitMs, 'auto-retry');
    return;
  }

  // Execute automation
  return await executeAutomation(automationId, triggerData);
});

// Queue events for monitoring
executionQueue.on('failed', (job, err) => {
  const connector = job.data.connector;
  recordFailure(connector);
  checkCircuitBreaker(connector);
});

executionQueue.on('completed', (job) => {
  const connector = job.data.connector;
  clearFailureCount(connector);  // Reset on success
});
```

### Circuit Breaker Pattern

```typescript
interface CircuitBreakerState {
  connectorId: string;
  failureCount: number;
  state: 'healthy' | 'degraded' | 'offline';
  lastFailureAt?: Date;
  nextRetryAt?: Date;
}

class CircuitBreaker {
  private states = new Map<string, CircuitBreakerState>();
  private failureThreshold = 5;
  private backoffIntervals = [60000, 300000, 900000, 3600000];  // 1m, 5m, 15m, 1h (ms)

  async recordFailure(connectorId: string) {
    let state = this.states.get(connectorId) || {
      connectorId,
      failureCount: 0,
      state: 'healthy'
    };

    state.failureCount += 1;
    state.lastFailureAt = new Date();

    if (state.failureCount >= this.failureThreshold) {
      state.state = 'offline';
      const backoffLevel = Math.min(state.failureCount - this.failureThreshold, this.backoffIntervals.length - 1);
      state.nextRetryAt = new Date(Date.now() + this.backoffIntervals[backoffLevel]);
    }

    this.states.set(connectorId, state);
    await db('circuit_breaker_states').insert(state).onConflict('connector_id').merge();
  }

  async recordSuccess(connectorId: string) {
    const state = this.states.get(connectorId);
    if (state) {
      state.failureCount = 0;
      state.state = 'healthy';
      state.nextRetryAt = undefined;
      this.states.set(connectorId, state);
      await db('circuit_breaker_states').update(state);
    }
  }

  async isCircuitOpen(connectorId: string): Promise<boolean> {
    const state = this.states.get(connectorId);
    if (!state || state.state === 'healthy') return false;

    if (state.state === 'offline' && state.nextRetryAt && new Date() >= state.nextRetryAt) {
      // Time to retry - close circuit temporarily
      state.state = 'degraded';
      return false;
    }

    return state.state === 'offline';
  }

  async resetCircuit(connectorId: string) {
    const state = this.states.get(connectorId);
    if (state) {
      state.failureCount = 0;
      state.state = 'healthy';
      this.states.set(connectorId, state);
      await db('circuit_breaker_states').update(state);
    }
  }
}
```

### Backpressure Handling

```typescript
// When executing automation
async function enqueueExecution(automationId: string, triggerData: any) {
  const queueDepth = await executionQueue.count();

  if (queueDepth >= MAX_QUEUE_SIZE) {
    // Return 429 Too Many Requests
    throw new Error('Queue full - try again later');
  }

  const job = await executionQueue.add(
    { automationId, triggerData },
    { attempts: 3, backoff: { type: 'exponential', delay: 2000 } }
  );

  return { jobId: job.id, queueDepth: queueDepth + 1 };
}

// API endpoint for queue status
router.get('/status/queue', async (req, res) => {
  const counts = await executionQueue.getJobCounts();
  const queueDepth = counts.waiting + counts.active;
  const oldestJob = await executionQueue.getOldest();

  res.json({
    queueDepth,
    activeCount: counts.active,
    waitingCount: counts.waiting,
    oldestItemAge: oldestJob ? Date.now() - oldestJob.timestamp : null,
    percentFull: (queueDepth / MAX_QUEUE_SIZE) * 100
  });
});
```
