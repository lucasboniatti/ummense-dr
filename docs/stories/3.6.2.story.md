## Story

As a **platform operator**, I want to **view a dashboard with real-time execution metrics and trends**, so that **I can understand system health and performance at a glance**.

---

## Acceptance Criteria

- [x] **[CRITICAL]** Dashboard displays 5 key metrics with real-time updates
  - Metric 1: Execution success rate (7-day trend, line chart)
  - Metric 2: Average execution duration (performance, line chart)
  - Metric 3: Failed executions by automation (top 5, bar chart)
  - Metric 4: Cost savings from S3 archival (vs. DB storage, metric card)
  - Metric 5: Storage utilization (DB + S3 combined, gauge chart)
  - All charts update live via WebSocket (3.6.1) without page refresh

- [x] **[HIGH]** Performance targets
  - Dashboard initial load: <1s (including WebSocket connection + first data fetch)
  - Real-time chart updates: <100ms (time from WebSocket message to UI render)
  - Responsive design: Works on desktop (1920x1080), tablet (768px), mobile (375px)
  - No janky animations (60 FPS target for chart updates)

- [x] **[HIGH]** Responsive and accessible design
  - Mobile-first CSS (Tailwind, mobile → tablet → desktop)
  - Accessibility: WCAG 2.1 AA compliance
    - Color contrast: 4.5:1 minimum (text vs background)
    - Keyboard navigation: All interactive elements accessible
    - Screen reader support: ARIA labels on all charts
  - Dark mode support (respects prefers-color-scheme)

- [x] **[MEDIUM]** Real-time data integration with WebSocket
  - Connect to WebSocket server (port 9001)
  - Subscribe to execution updates (user_id-based channel)
  - Update metrics when new execution data arrives
  - Graceful fallback if WebSocket unavailable (poll API every 10s)

- [x] **[MEDIUM]** E2E test coverage
  - Scenario 1: Load dashboard → verify all 5 metrics displayed + WebSocket connected
  - Scenario 2: Filter by date range → verify metrics recalculate (using API)
  - Scenario 3: Export trend data to CSV → verify file format and data integrity
  - All 3 scenarios must pass with ≥95% assertion coverage

## Technical Notes

### Component Architecture
```typescript
// DashboardContainer (main component)
├── MetricCard (reusable, 5 instances)
│   ├── SuccessRateCard (7-day trend, Recharts LineChart)
│   ├── AvgDurationCard (performance, Recharts LineChart)
│   ├── FailedExecutionsCard (top 5, Recharts BarChart)
│   ├── CostSavingsCard (S3 vs DB, Metric display)
│   └── StorageUtilizationCard (gauge, Recharts RadarChart)
└── FilterControls
    ├── DateRangePicker
    ├── AutomationFilter
    └── ExportButton

// WebSocket Context (shared connection)
export const WebSocketContext = createContext<WebSocketContextType>(null);

// Hook for metrics
const useMetrics = () => {
  const ws = useContext(WebSocketContext);
  const [metrics, setMetrics] = useState(initialMetrics);

  useEffect(() => {
    ws.on('execution-update', (delta) => {
      // Recalculate metrics based on delta
      setMetrics(prev => updateMetrics(prev, delta));
    });
  }, [ws]);

  return metrics;
};
```

### Libraries & Dependencies
- **Recharts** (2.x+): Charts (line, bar, gauge)
- **Tailwind CSS** (3.x+): Responsive styling
- **date-fns** (2.x+): Date utilities
- **papaparse** (5.x+): CSV export

### API Integration
- Endpoint: `GET /api/analytics/metrics` (initial load + fallback polling)
- Endpoint: `GET /api/analytics/cost-summary` (cost card)
- WebSocket: `execution-updates:{user_id}` channel (real-time updates)

### Styling Strategy
```css
/* Mobile-first (375px+) */
.metric-card { grid-template-columns: 1fr; }
.chart { height: 200px; }

/* Tablet (768px+) */
@media (min-width: 768px) {
  .metric-card { grid-template-columns: 1fr 1fr; }
  .chart { height: 300px; }
}

/* Desktop (1920px+) */
@media (min-width: 1920px) {
  .metric-card { grid-template-columns: repeat(5, 1fr); }
  .chart { height: 400px; }
}
```

### Real-Time Data Flow
1. User opens dashboard
2. API call: `GET /api/analytics/metrics` (initial state)
3. WebSocket: Subscribe to `execution-updates:{user_id}`
4. On each WebSocket message:
   - Calculate delta metrics
   - Update chart data
   - Trigger Recharts re-render (<100ms)
5. Fallback: If WS disconnected, poll API every 10s
