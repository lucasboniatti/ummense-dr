## Story

As a **user**, I want to **ver minha localização atual na hierarquia de navegação através de breadcrumbs**, so that **eu possa entender onde estou e navegar rapidamente para níveis superiores**.

---

## Acceptance Criteria

- [ ] Componente Breadcrumb existente é consolidado como primitive compartilhado da aplicação.
- [ ] Exibe hierarquia de navegação: Home > Seção > Página atual.
- [ ] Página atual não é clicável (apenas texto).
- [ ] Itens anteriores são links clicáveis.
- [ ] Separador visual entre itens (chevron ou slash).
- [ ] Colapsa em mobile (exibe apenas último nível).
- [ ] Acessível com nav e aria-label="Breadcrumb".
- [ ] Dark mode suportado.

## Dev Notes

### Relevant Context

- O frontend já possui um componente `Breadcrumb`, mas ele ainda não está posicionado como primitive compartilhado para a navegação do shell.
- Esta story deve estabilizar o componente base e a integração inicial; a resolução dinâmica mais rica fica na `10.16`.

### API Design

```tsx
import { Breadcrumb, BreadcrumbItem } from '@/components/ui/Breadcrumb';

<Breadcrumb>
  <BreadcrumbItem href="/">Dashboard</BreadcrumbItem>
  <BreadcrumbItem href="/dashboard/automations">Fluxos</BreadcrumbItem>
  <BreadcrumbItem current>Detalhe</BreadcrumbItem>
</Breadcrumb>
```

### Resolução Dinâmica

```tsx
// utils/breadcrumbs.ts
const routeMap: Record<string, string> = {
  '/': 'Painel',
  '/dashboard': 'Painel',
  '/dashboard/automations': 'Fluxos',
  '/dashboard/webhooks': 'Webhooks',
  '/cards': 'Cards',
};

export function resolveBreadcrumbs(pathname: string): BreadcrumbItem[] {
  // Split pathname and map to labels
}
```

- Reutilizar o componente existente em vez de criar uma segunda implementação.
