## Story

As a **user**, I want to **confirmar ações destrutivas através de um diálogo de confirmação**, so that **eu não delete ou altere dados importantes por acidente**.

---

## Acceptance Criteria

- [ ] Componente `ConfirmDialog` existente é consolidado como primitive oficial para confirmações destrutivas.
- [ ] Props: title, description, confirmLabel, cancelLabel, variant (danger/warning/info).
- [ ] Botão de confirmar tem cor diferente para ações destrutivas (vermelho).
- [ ] Focus trap no diálogo.
- [ ] ESC fecha o diálogo.
- [ ] Click fora fecha o diálogo (configurável).
- [ ] Acessível com role="alertdialog".
- [ ] Hook `useConfirmDialog` para uso programático.
- [ ] Dark mode suportado.

## Dev Notes

### Relevant Context

- O frontend já possui `ConfirmDialog` e `useConfirmDialog` no mesmo módulo.
- Há consumidores atuais em webhooks, histórico de execuções e controles operacionais.
- O objetivo desta story e endurecer o primitive e estabilizar o contrato, nao duplicar implementação.

### API Design

```tsx
import { ConfirmDialog, useConfirmDialog } from '@/components/ui/ConfirmDialog';

// Declarative
<ConfirmDialog
  open={isOpen}
  onOpenChange={setIsOpen}
  title="Excluir tarefa"
  description="Esta ação não pode ser desfeita. Tem certeza?"
  confirmLabel="Excluir"
  cancelLabel="Cancelar"
  variant="danger"
  onConfirm={handleDelete}
/>

// Imperative (hook)
const { confirm } = useConfirmDialog();

const handleDelete = async () => {
  const confirmed = await confirm({
    title: "Excluir tarefa",
    description: "Esta ação não pode ser desfeita.",
    variant: "danger",
  });
  if (confirmed) {
    // execute delete
  }
};
```

### Relevant Source Tree

- `packages/frontend/src/components/ui/ConfirmDialog.tsx`
- `packages/frontend/src/pages/dashboard/webhooks/index.tsx`
- `packages/frontend/src/components/ExecutionHistoryTable.tsx`
- `packages/frontend/src/components/PauseResumeControls.tsx`

### Implementation Guidance

- Tratar `ConfirmDialog.tsx` como fonte única de verdade para a API de confirmação.
- Não mover o hook para outro arquivo a menos que isso aconteça sem criar API duplicada ou import path concorrente.
- Registrar claramente o contrato que será reutilizado pela `10.13`.

### Evidence Expectations

- Lista dos consumidores atuais alinhados ao contrato final.
- Evidência de `alertdialog`, ESC, outside click e dark mode.
