## Story

As a **platform operator**, I want to **save and reuse filter combinations for execution history queries**, so that **I can quickly find the same execution patterns without redefining filters each time**.

---

## Acceptance Criteria

- [x] **[HIGH]** Users can save current filter state as a preset
  - Click "Save as Preset" button in ExecutionHistoryTable filter bar
  - Dialog: Enter preset name + optional description
  - Preset stored in `user_saved_filters` table (user_id, name, description, filter_json)
  - Max 20 presets per user (enforced in API)

- [x] **[HIGH]** Users can apply saved preset with single click
  - Dropdown: "Load Preset" shows list of user's saved presets
  - Click preset → Load filters + re-run search (via searchExecutionHistory API)
  - Performance: Apply preset <500ms (reuses existing API)

- [x] **[MEDIUM]** Users can manage presets (edit, delete)
  - Delete preset: Soft-delete (preserve audit trail, set deleted_at timestamp)
  - Edit: Not required (delete old, save new)
  - Presets don't clutter UI: Show only active (deleted_at IS NULL)

- [x] **[MEDIUM]** Pre-populate 3 default presets (non-deletable)
  - Preset 1: "Failed Executions (24h)" — status=failed, date_range=last 24h
  - Preset 2: "Timeout Errors" — search_term=timeout, status=failed
  - Preset 3: "Recent Executions" — status=any, date_range=last 7 days, sort=newest

- [x] **[MEDIUM]** API endpoints (CRUD)
  - POST `/api/users/saved-filters` — Create preset
  - GET `/api/users/saved-filters` — List user's presets
  - DELETE `/api/users/saved-filters/:id` — Delete preset
  - All endpoints: User-scoped (RLS policy enforces)

## Technical Notes

### Database Schema

```sql
CREATE TABLE user_saved_filters (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  name TEXT NOT NULL,
  description TEXT,
  filter_json JSONB NOT NULL, -- {searchTerm, automationId, status, dateRange}
  is_default BOOLEAN DEFAULT false, -- true for pre-populated presets
  created_at TIMESTAMP DEFAULT now(),
  deleted_at TIMESTAMP, -- Soft delete

  -- Active presets remain unique per user; soft-deleted names can be reused
  UNIQUE(user_id, name)
);

-- RLS Policy: Users can only see own presets
ALTER TABLE user_saved_filters ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can only see own presets"
  ON user_saved_filters FOR SELECT
  USING (auth.uid() = user_id);

-- Index for common queries
CREATE INDEX idx_saved_filters_user_active
  ON user_saved_filters(user_id, deleted_at);
```
