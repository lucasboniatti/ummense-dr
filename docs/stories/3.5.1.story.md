### Stack Trace Sanitization

```typescript
// Sanitization patterns for removing sensitive data
const SENSITIVE_PATTERNS = [
  /(\b[Pp]assword\s*[=:]\s*)['\"]?([^'"\s,;]+)['\"]?/g,
  /(\b[Tt]oken\s*[=:]\s*)['\"]?([^'"\s,;]+)['\"]?/g,
  /([Aa]pi[_-]?[Kk]ey\s*[=:]\s*)['\"]?([^'"\s,;]+)['\"]?/g,
  /([Ss]ecret\s*[=:]\s*)['\"]?([^'"\s,;]+)['\"]?/g,
  /(Bearer\s+)[^\s,;]+/g,
  /(Authorization:\s+)[^\s,;]+/g,
  /\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b/g,  // email
  /\b\d{3}[-.]?\d{3}[-.]?\d{4}\b/g,  // phone
  /\b\d{3}[-]?\d{2}[-]?\d{4}\b/g     // SSN
];

function sanitizeStackTrace(errorContext: any): any {
  const sanitized = { ...errorContext };

  if (sanitized.stack_trace) {
    let cleaned = sanitized.stack_trace;
    SENSITIVE_PATTERNS.forEach(pattern => {
      cleaned = cleaned.replace(pattern, '[REDACTED]');
    });
    sanitized.stack_trace = cleaned;
  }

  if (sanitized.error_message) {
    let cleaned = sanitized.error_message;
    SENSITIVE_PATTERNS.forEach(pattern => {
      cleaned = cleaned.replace(pattern, '[REDACTED]');
    });
    sanitized.error_message = cleaned;
  }

  return sanitized;
}
```

### S3 Archival Extension

```typescript
async function archiveToS3(executions: any[], bucket: string, userId: string) {
  const archiveDate = new Date().toISOString().split('T')[0];  // YYYY-MM-DD
  const key = `archive/${userId}/${archiveDate}/execution-history.json.gz`;

  const s3 = new AWS.S3();
  const jsonData = JSON.stringify(executions, null, 2);
  const gzipped = await gzip(jsonData);

  try {
    await s3.putObject({
      Bucket: bucket,
      Key: key,
      Body: gzipped,
      ServerSideEncryption: 'AES256'
    }).promise();

    return { success: true, key, recordCount: executions.length };
  } catch (error) {
    console.error(`Failed to archive to S3: ${error}`);
    throw new Error(`S3 archival failed for user ${userId}: ${error.message}`);
  }
}
```

### Full-Text Search Implementation

```typescript
// In ExecutionHistoryService
async function searchExecutionHistory(query: {
  userId: string;
  searchTerm: string;
  filters?: { automationId?: string; dateRange?: [Date, Date]; status?: string };
  limit?: number;
  offset?: number;
}) {
  let q = db('automation_executions')
    .where('user_id', query.userId)
    .select('*');

  // Full-text search using tsvector
  if (query.searchTerm) {
    q = q.whereRaw(`search_vector @@ plainto_tsquery('english', ?)`, [query.searchTerm]);
  }

  // Apply optional filters
  if (query.filters?.automationId) {
    q = q.where('automation_id', query.filters.automationId);
  }

  if (query.filters?.dateRange) {
    q = q.whereBetween('created_at', query.filters.dateRange);
  }

  if (query.filters?.status) {
    q = q.where('status', query.filters.status);
  }

  // Order by relevance (ts_rank) then recency
  q = q
    .select(db.raw('ts_rank(search_vector, plainto_tsquery(\'english\', ?)) as relevance', [query.searchTerm]))
    .orderBy('relevance', 'desc')
    .orderBy('created_at', 'desc')
    .limit(query.limit || 50)
    .offset(query.offset || 0);

  return q;
}
```

- ✅ **Performance Benchmarks**: Document actual latency/throughput at production scale

### Key Architectural Patterns

**1. Full-Text Search via PostgreSQL tsvector**
- Follows observability best practices from Story 3.1 (efficient querying at scale)
- Uses native PostgreSQL GIN indexes for sub-100ms search on 100K+ records
- Integrates with existing ExecutionHistoryService API

**2. Data Sanitization (Security by Default)**
- Scrubber runs BEFORE data enters database (shift-left security)
- Regex patterns detect: API keys, auth tokens, passwords, PII (emails, phones, SSN)
- Replaces sensitive values with `[REDACTED]` placeholder
- Applied to: error_context, error_message, stack traces in audit_logs

**3. S3 Archival (Cost & Compliance Optimization)**
- Extends Story 3.5 cleanup job to archive BEFORE delete (immutability + cost savings)
- Uses S3 Intelligent-Tiering for automatic cost optimization (warm → cold transition)
- Maintains audit trail compliance: Original records in DB (recent), archive in S3 (long-term)
- User-configurable: `user_retention_policies.archive_enabled` flag

**4. Performance Testing (Data-Driven Decisions)**
- Benchmarks validate Story 3.5 assumptions about query performance
- Test harness simulates production scale: 100K-500K execution records
- Documents actual latency/throughput vs. theoretical expectations

**Database & Search**
- PostgreSQL tsvector column on `automation_executions` and `audit_logs` for full-text search
- GIN indexes for fast phrase matching (<100ms on 100K+ records)
- Trigger/application layer updates search_vector on insert/update

**Stack Trace Sanitization**
- Sensitive patterns: passwords, API keys, tokens, auth headers, emails, phone numbers, SSN
- Implementation: Regex scrubber in error_context capture middleware
- Applied: Before database insert (immutable; cannot be removed later)
- Compliance: Prevents accidental PII leakage in CSV/JSON exports
- Cost: Minimal (~1-2ms overhead per error capture)

**S3 Archival Integration**
- Pre-requisite: AWS SDK configured with S3 bucket + IAM permissions
- Archive path: `s3://{bucket}/archive/{user_id}/{YYYY-MM-DD}/execution-history.json.gz`
- Compression: Gzip (typical 80-90% size reduction)
- Retention: 7 years (2555 days) per compliance requirements
- Cost model: S3 storage (~$0.023/GB/month) vs. PostgreSQL (~$1-5/GB/month) = 50-200x cheaper

- Benchmarks: Measure latency for typical queries (history list, detail page, export, search)
- Tools: Node.js Autocannon or Artillery for concurrent load testing
