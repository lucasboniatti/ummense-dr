## Story

As a **user**, I want to **manage webhooks through an intuitive dashboard** so that **I can easily create, test, view delivery history, and debug webhooks without technical barriers**.

- [ ] **[CRITICAL - Architect]** URL validation UI-side: Real-time validation feedback (valid/invalid) before save
- [ ] **[CRITICAL - Architect]** API key security: Generated keys displayed only once, masked in edit view (show last 4 chars only)
- [ ] Webhook management page created at `/dashboard/webhooks`:
  - List view: Table with webhook URL, status (enabled/disabled), last triggered, delivery success rate
  - Create/edit modal: URL input, API key generation, description, active toggle
  - Delete confirmation: Soft delete, keep history
- [ ] Webhook detail view:
  - Delivery history: Last 50 deliveries with status, attempt count, timestamp, response status code
  - Live delivery graph: Success/failure rate over last 24 hours
  - Test payload sender: Manual trigger with sample event data
  - Response inspector: View request headers, response status, response body
- [ ] **[HIGH - Architect]** Test webhook feature:
  - Pre-populate with sample event (task:created, rule:executed, webhook:triggered)
  - Send test payload immediately (bypass queue)
  - Display response in real-time (status, headers, body)
  - Show elapsed time for request

  - Filter by status (success, failed, pending, dead-lettered)
  - Filter by date range (last 24h, 7d, 30d, custom)

## Technical Notes

### URL Validation Rules (Client-Side)
```javascript
const urlValidation = (url) => {
  // 1. Must be HTTPS
  if (!url.startsWith('https://')) return false;

  // 2. Valid URL format
  try { new URL(url); } catch { return false; }

  // 3. Not localhost or private IP (SSRF check)
  const hostname = new URL(url).hostname;
  const blocklist = ['localhost', '127.0.0.1', ...];
  if (blocklist.includes(hostname)) return false;

  return true;
};
```

### API Key Display Pattern
```javascript
// Generation
const apiKey = generateRandomKey(32); // Full key
const displayKey = `sk_${apiKey.slice(-4)}`;

// Storage

// Display in UI
<input value={displayKey} disabled /> // Only show last 4
```

### Delivery History Query Performance
```sql
-- Indexed for fast filtering
SELECT * FROM webhook_deliveries
WHERE webhook_id = $1
AND created_at BETWEEN $2 AND $3
AND status IN ('success', 'failed', 'dead_lettered', 'pending')
ORDER BY created_at DESC
LIMIT 50;

-- Indexes:
CREATE INDEX idx_webhook_deliveries_webhook_created
ON webhook_deliveries(webhook_id, created_at DESC);
CREATE INDEX idx_webhook_deliveries_status
ON webhook_deliveries(webhook_id, status);
```
