## Story

As an **operations manager**, I want to **gerenciar meus fluxos em múltiplas visões (quadro, tabela e indicadores) com persistência real**, so that **eu acompanhe execução e capacidade operacional em tempo real**.

---

## Acceptance Criteria

- [ ] Rota de fluxos apresenta abas `Quadro`, `Tabela` e `Indicadores` para o mesmo contexto de dados.
- [ ] Quadro kanban exibe colunas com contagem e cards com metadados operacionais (responsáveis, progresso, status).
- [ ] Drag-and-drop de card entre colunas persiste no backend e reflete imediatamente na UI.
- [ ] Filtro por texto/status funciona nas três visões.
- [ ] Visão de indicadores exibe métricas mínimas por fluxo (cards ativos, concluídos, bloqueados, throughput).
- [ ] Dados não dependem de mock local para operação principal.
- [ ] Performance mínima: render inicial do board <= 2.0s (dataset de referência: 500 cards).
- [ ] Performance mínima: persistência de drag-and-drop <= 800ms p95 (UI + API roundtrip).
- [ ] Performance mínima: aplicação de filtro <= 300ms p95 nas 3 visões.

---

## Tasks / Subtasks

- [ ] Definir UX e comportamento das 3 visões de fluxo (AC: 1, 2, 4, 5).
- [ ] Implementar board operacional com DnD persistido (AC: 2, 3, 6).
- [ ] Implementar visão tabela para gestão em massa (AC: 1, 4).
- [ ] Implementar visão indicadores conectada a métricas reais (AC: 1, 5, 6).
- [ ] Integrar filtros unificados e estado compartilhado entre views (AC: 4).

---

## Testing

- [ ] E2E: mover card entre colunas e validar persistência após refresh.
- [ ] E2E: alternar quadro/tabela/indicadores sem perda de filtro.
- [ ] API contract tests para endpoints de fluxos e cards.
- [ ] Performance test: medir render inicial, DnD persistido e filtro com dataset padrão (500 cards).
