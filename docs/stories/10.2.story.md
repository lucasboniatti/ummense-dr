## Dev Notes

### Mapeamento de Emojis para Lucide

| Emoji | Lucide Icon |
|-------|-------------|
| ✓ | `Check` |
| ⏱ | `Clock` |
| ✗ | `X` |
| 💾 | `Database` |
| ⚠️ | `AlertTriangle` |
| ℹ️ | `Info` |
| ❌ | `XCircle` |

### Relevant Context

- `MetricCard.tsx` ja aceita `ReactNode` para `icon`; a story nao deve reverter esse contrato nem recria-lo.
- Os remanescentes mais provaveis de emoji estao em paginas de integracoes e banners informativos, nao apenas em KPI cards.
- A auditoria deve focar em iconografia decorativa e de status visivel ao operador.

### Arquivo de referência: `MetricCard.tsx`

```tsx
// Contrato atual já suportado:
interface MetricCardProps {
  icon?: React.ReactNode;
}
```

### Relevant Source Tree

- `packages/frontend/src/components/MetricCard.tsx`
- `packages/frontend/src/components/KPICards.tsx`
- `packages/frontend/src/pages/dashboard/integrations/index.tsx`
- `packages/frontend/src/pages/auth/integration-callback.tsx`

### Implementation Guidance

- Executar uma auditoria com `rg` no frontend antes de alterar componentes.
- Priorizar superfícies operacionais e banners com emoji usados como icone.
- Nao substituir setas de tendencia ou caracteres textuais que nao sejam iconografia visual principal sem necessidade comprovada.

### Evidence Expectations

- Lista curta dos arquivos auditados.
- Before/after das principais superfícies corrigidas.
