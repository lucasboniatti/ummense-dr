# Story 1.1: Autenticação com JWT e Bcrypt

**Epic:** 1.0 (MVP Foundation - Wave 1)
**Status:** Done
**Priority:** P1 (Blocker)
**Points:** 8

## Story
Como desenvolvedor do TaskFlow, preciso de um sistema de autenticação robusto com JWT e Bcrypt para garantir que apenas usuários autenticados possam acessar o aplicativo.

## Acceptance Criteria

### AC-1: Endpoint POST /auth/signup
- [x] Aceita email e password
- [x] Valida email (formato RFC)
- [x] Valida password (min 8 chars, letter + number)
- [x] Verifica se email já existe
- [x] Hash password com bcrypt (salt rounds=10)
- [x] Cria usuário no banco
- [x] Retorna JWT token (24h expiration)
- [x] Define cookie httpOnly com token
- [x] Retorna 201 Created com user object

### AC-2: Endpoint POST /auth/login
- [x] Aceita email e password
- [x] Busca usuário por email
- [x] Compara password com hash bcrypt
- [x] Gera JWT token (24h)
- [x] Define cookie httpOnly
- [x] Retorna 200 OK com user object
- [x] Retorna 401 Unauthorized se credenciais inválidas
- [x] Não revela se email existe (security)

### AC-3: Endpoint GET /auth/me
- [x] Requer JWT válido via cookie ou Authorization header
- [x] Retorna dados do usuário autenticado
- [x] Retorna 401 se token inválido/expirado

### AC-4: Endpoint POST /auth/logout
- [x] Limpa cookie de token
- [x] Retorna 200 OK

### AC-5: Middleware de Autenticação
- [x] Verifica JWT em cookies ou Authorization header
- [x] Valida signature JWT
- [x] Valida expiração
- [x] Injeta user no contexto request
- [x] Retorna 401 se inválido

### AC-6: Security Measures
- [x] Bcrypt salt rounds = 10
- [x] JWT expiration = 24 horas
- [x] Cookies httpOnly (não acessível via JS)
- [x] CORS headers configurado
- [x] Password min 8 chars, letter + number
- [x] Email validation RFC 5322

## Technical Details

### Database Schema (Migration 001)
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_email ON users(email);
```

### API Routes

**POST /auth/signup**
```
Request: { email, password }
Response: { id, email, token }
```

**POST /auth/login**
```
Request: { email, password }
Response: { id, email, token }
```

**GET /auth/me**
```
Headers: Authorization: Bearer <token> OR Cookie: token=<token>
Response: { id, email }
```

**POST /auth/logout**
```
Response: { message: "Logged out" }
```

### Technology Stack
- **Password:** bcrypt (salt rounds=10)
- **JWT:** jsonwebtoken (HS256, 24h expiration)
- **Cookie:** httpOnly, secure
- **Validation:** email-validator, zod
