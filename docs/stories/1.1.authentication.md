# Story 1.1: Autenticação com JWT e Bcrypt

**Epic:** 1.0 (MVP Foundation - Wave 1)
**Status:** Ready for Review
**Priority:** P1 (Blocker)
**Points:** 8

## Story
Como desenvolvedor do TaskFlow, preciso de um sistema de autenticação robusto com JWT e Bcrypt para garantir que apenas usuários autenticados possam acessar o aplicativo.

## Acceptance Criteria

### AC-1: Endpoint POST /auth/signup
- [ ] Aceita email e password
- [ ] Valida email (formato RFC)
- [ ] Valida password (min 8 chars, letter + number)
- [ ] Verifica se email já existe
- [ ] Hash password com bcrypt (salt rounds=10)
- [ ] Cria usuário no banco
- [ ] Retorna JWT token (24h expiration)
- [ ] Define cookie httpOnly com token
- [ ] Retorna 201 Created com user object

### AC-2: Endpoint POST /auth/login
- [ ] Aceita email e password
- [ ] Busca usuário por email
- [ ] Compara password com hash bcrypt
- [ ] Gera JWT token (24h)
- [ ] Define cookie httpOnly
- [ ] Retorna 200 OK com user object
- [ ] Retorna 401 Unauthorized se credenciais inválidas
- [ ] Não revela se email existe (security)

### AC-3: Endpoint GET /auth/me
- [ ] Requer JWT válido via cookie ou Authorization header
- [ ] Retorna dados do usuário autenticado
- [ ] Retorna 401 se token inválido/expirado

### AC-4: Endpoint POST /auth/logout
- [ ] Limpa cookie de token
- [ ] Retorna 200 OK

### AC-5: Middleware de Autenticação
- [ ] Verifica JWT em cookies ou Authorization header
- [ ] Valida signature JWT
- [ ] Valida expiração
- [ ] Injeta user no contexto request
- [ ] Retorna 401 se inválido

### AC-6: Security Measures
- [ ] Bcrypt salt rounds = 10
- [ ] JWT expiration = 24 horas
- [ ] Cookies httpOnly (não acessível via JS)
- [ ] CORS headers configurado
- [ ] Password min 8 chars, letter + number
- [ ] Email validation RFC 5322

## Technical Details

### Database Schema (Migration 001)
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_email ON users(email);
```

### API Routes

**POST /auth/signup**
```
Request: { email, password }
Response: { id, email, token }
```

**POST /auth/login**
```
Request: { email, password }
Response: { id, email, token }
```

**GET /auth/me**
```
Headers: Authorization: Bearer <token> OR Cookie: token=<token>
Response: { id, email }
```

**POST /auth/logout**
```
Response: { message: "Logged out" }
```

### Technology Stack
- **Password:** bcrypt (salt rounds=10)
- **JWT:** jsonwebtoken (HS256, 24h expiration)
- **Cookie:** httpOnly, secure
- **Validation:** email-validator, zod
