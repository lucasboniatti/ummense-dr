## Story

As a **user**, I want to **integrate with Slack and Discord through OAuth** so that **I can send notifications and execute commands directly from automations without managing tokens manually**.

## Acceptance Criteria

- [ ] **[CRITICAL - Architect]** OAuth PKCE implementation (no client secret exposure):
  - Authorization Code Flow with PKCE (RFC 7636)
  - code_challenge = BASE64URL(SHA256(code_verifier))
  - code_challenge_method = "S256"
  - Exchange code for token on backend only
- [ ] **[CRITICAL - Architect]** Token encryption via AWS KMS:
  - Token storage: encrypted_token (AES-256), kms_key_id for decryption
  - Key rotation: supported (multiple kms_key_id values)
  - Never log or return plain tokens
- [ ] **[CRITICAL - Architect]** Rate limiting per integration:
  - Slack: 60 requests per minute (per workspace)
  - Discord: 50 requests per minute (per server)
  - Backoff when rate-limited (retry after X seconds header)
- [ ] Slack integration (post messages to channels/DMs):
  - OAuth scope: chat:write, chat:write.public
  - Send message action: channel, thread_ts, text, blocks (formatted)
  - Send file upload action: channel, file content, title
  - List channels action: Dropdown for user to select target channel
- [ ] Discord integration (post messages to channels):
  - OAuth scope: chat.write (Discord v14+)
  - Send message action: channel_id, message (content + embeds)
  - Send file upload action: channel_id, file + description
  - List channels action: Dropdown for user to select target channel
- [ ] **[HIGH - Architect]** Webhook actions for Slack/Discord:
  - Trigger: automation:executed, rule:completed
  - Action: send_slack_message, send_discord_message
  - Template support: {{variable}} substitution (rule_name, status, error, duration_ms)
  - Example: "Rule {{rule_name}} completed in {{duration_ms}}ms with status {{status}}"
- [ ] **[HIGH - Architect]** Slash command support (Slack):
  - Command: `/automation` to trigger rule from Slack
  - Respond with rule execution status
  - Support one-off execution without webhook setup
  - Rate limit: 3 per minute per workspace
- [ ] Integration token management:
  - Disconnect integration: Revoke token, delete from DB
  - Token refresh: Auto-refresh before expiration
  - Token expiration monitoring: Alert if token expires in <7 days
- [ ] Test coverage:
  - OAuth PKCE flow validates code_challenge
  - Token encryption/decryption works correctly
  - Rate limiting blocks requests after limit exceeded
  - Message templates substitute variables correctly
  - Slash command executes rules and responds
  - File uploads work for both Slack and Discord
  - Token refresh triggers before expiration
  - Disconnecting revokes token and removes DB entry

## Technical Notes

### OAuth PKCE Flow

**Step 1: Generate code_verifier + code_challenge (Frontend)**
```javascript
const codeVerifier = generateRandomString(128); // 128 chars
const codeChallenge = BASE64URL(SHA256(codeVerifier));

// Store in sessionStorage (client-side only)
sessionStorage.setItem('code_verifier', codeVerifier);
sessionStorage.setItem('code_challenge', codeChallenge);

// Redirect to OAuth provider
window.location = `https://slack.com/oauth/v2/authorize?client_id=XXX&code_challenge=${codeChallenge}&code_challenge_method=S256`;
```

**Step 2: Exchange code for token (Backend)**
```javascript
// User redirected back with authorization code
const code = req.query.code;
const codeVerifier = req.session.code_verifier; // From session

const response = await fetch('https://slack.com/api/oauth.v2.access', {
  method: 'POST',
  body: {
    client_id: process.env.SLACK_CLIENT_ID,
    // NO client_secret here - PKCE replaces it
    code: code,
    code_verifier: codeVerifier  // Prove we generated the challenge
  }
});

const token = response.access_token;

// Encrypt and store
const encryptedToken = encrypt(token, kmsKeyId);
await db.slackTokens.create({
  user_id: userId,
  workspace_id: response.team_id,
  encrypted_token: encryptedToken,
  kms_key_id: kmsKeyId,
  token_type: 'bot'
});
```

### Token Encryption (KMS)
```javascript
const AWS = require('aws-sdk');
const kms = new AWS.KMS();

// Encrypt
const encryptedToken = await kms.encrypt({
  KeyId: process.env.KMS_KEY_ID,
  Plaintext: plainToken
}).promise();

// Decrypt (when needed to make API call)
const decryptedToken = await kms.decrypt({
  CiphertextBlob: encryptedToken
}).promise();

// Key rotation: Track which key was used, support multiple keys
```

### Slack Message Template
```javascript
const template = "Rule {{rule_name}} executed in {{duration_ms}}ms - Status: {{status}}";
const variables = {
  rule_name: 'High Priority Task Alert',
  duration_ms: 245,
  status: 'success'
};

const message = template.replace(/\{\{(\w+)\}\}/g, (match, key) => variables[key]);
// Output: "Rule High Priority Task Alert executed in 245ms - Status: success"
```

### Rate Limiting
```javascript
// Redis counter: "{integration}:{workspace_id}:requests"
const key = `slack:${workspace_id}:requests`;
const count = await redis.incr(key);
if (count === 1) await redis.expire(key, 60); // Reset every minute

if (count > 60) {
  // Rate limited
  const retryAfter = await redis.ttl(key);
  throw new Error(`Rate limited. Retry after ${retryAfter}s`);
}
```

### Slash Command (Slack)
```javascript
// Webhook endpoint: POST /slack/slash-commands
app.post('/slack/slash-commands', async (req, res) => {
  const { command, text, user_id, team_id } = req.body;

  if (command === '/automation') {
    const ruleId = text.split(' ')[0];
    const result = await executeRule(ruleId, { source: 'slack_slash' });

    res.json({
      response_type: 'in_channel',
      text: `Rule execution started. Check dashboard for status.`,
      blocks: [
        { type: 'section', text: { type: 'mrkdwn', text: `*Status:* ${result.status}` } }
      ]
    });
  }
});
```

## Dev Notes

- OAuth tokens stored encrypted, never logged or returned plaintext
- PKCE replaces client_secret (safer for OAuth flows)
- Token refresh automatic (triggered by 401 response)
- Slash commands use request signing for security (validate X-Slack-Signature)
- Rate limiting per workspace (shared pool for all users in workspace)
- Template variables: rule_name, status, error_message, duration_ms, executed_at
- Disconnection revokes token (call Slack/Discord revoke endpoint)
