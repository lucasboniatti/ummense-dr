## Dev Notes

### Relevant Source Tree

- Root currently centralizes docs and orchestration (`AGENTS.md`, `.aios-core/`, `docs/`, `packages/`).
- Backend source exists at `packages/backend/src/*` with runtime entrypoints already present.
- Frontend source exists at `packages/frontend/src/*` but package manifest is missing.

### Baseline Constraints from Current Repository

- Quality gates required by project rules: `npm run lint`, `npm run typecheck`, `npm test`.
- Root currently does not expose these scripts through a workspace manifest.
- `packages/frontend/package.json` must be created for local orchestration.
- `packages/backend/package.json` must be expanded with runnable scripts (today it is dependency-only).

### Implementation Notes for @dev

- Keep script names canonical and consistent across root/frontend/backend.
- Avoid introducing custom command names that bypass mandated quality gates.
- Root execution must be the default developer path; no manual `cd` workflow as primary path.

### Testing Standards

- Validate both positive path (all commands pass) and failure path (misconfiguration yields actionable error).
- Record executed commands and outputs in Dev Agent Record for QA replay.
