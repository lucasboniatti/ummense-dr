## Story

As an **executor user**, I want to **abrir e editar tarefas em um modal rico com histórico de alterações**, so that **eu tenha rastreabilidade e controle fino de execução por tarefa**.

---

## Acceptance Criteria

- [x] Modal de tarefa exibe título, descrição, tags, prioridade, card vinculado, responsável e prazo.
- [x] Prioridade e status possuem controles claros e consistentes com regras de negócio.
- [x] Histórico da tarefa registra criação, alterações de campo e mudanças de responsável/status.
- [x] Modal pode ser aberto a partir de painel, card e fluxo sem inconsistência de dados.
- [x] Falhas de validação exibem mensagens claras e não quebram a navegação.

---

## Tasks / Subtasks

- [x] Definir UX do modal e estados de interação (AC: 1, 2, 5).
- [x] Implementar formulário completo de tarefa com validações (AC: 1, 2, 5).
- [x] Implementar histórico/auditoria de alterações por tarefa (AC: 3).
- [x] Integrar pontos de abertura (painel/card/fluxo) com payload consistente (AC: 4).
- [x] Garantir contrato backend/frontend para update parcial e concorrência básica (AC: 2, 4).

---

## Testing

- [x] Unit tests de validação de formulário e regras de prioridade/status.
- [x] E2E: editar tarefa via modal em três entradas (painel/card/fluxo).
- [x] E2E: histórico da tarefa cresce com cada alteração relevante.
