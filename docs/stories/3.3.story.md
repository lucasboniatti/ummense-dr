## Story

## Technical Notes

### Scheduler Architecture

```
Scheduler service runs every 60 seconds:
  ↓
FOR EACH automation with trigger_type = 'scheduled':
  1. Parse cron expression
  2. Get current time in user's timezone
  3. Check if execution is due (cron(now) == true)
  4. If due:
     - Trigger execution via ExecutionService (Story 3.1)
     - Record scheduled_time (cron's intended time)
     - Record execution_time (actual time)
     - Record drift (difference)
     - Log in audit_logs
  ↓
If scheduler unavailable (downtime):
  - Log skipped execution attempt
  - Don't retry (no catch-up)
```

### Cron Parser & Scheduler Implementation

```typescript
import { CronExpression, CronDate } from 'cron-parser';  // npm package

interface ScheduledAutomation {
  automationId: string;
  cronExpression: string;
  timezone: string;
  enabled: boolean;
  lastExecutionAt: Date;
  nextExecutionAt: Date;
}

class SchedulerService {
  async evaluateCron(
    cronExpr: string,
    timezone: string,
    now: Date
  ): Promise<boolean> {
    try {
      const interval = new CronExpression(cronExpr, { tz: timezone });
      const nextExecution = interval.next().toDate();  // Next scheduled time

      // Check if we're within 60 seconds (±30) of scheduled time
      const diff = Math.abs(now.getTime() - nextExecution.getTime());
      return diff <= 60000;  // Within 1 minute window
    } catch (e) {
      throw new Error(`Invalid cron expression: ${cronExpr}`);
    }
  }

  async getNextExecutions(
    cronExpr: string,
    timezone: string,
    count: number = 3
  ): Promise<Date[]> {
    const interval = new CronExpression(cronExpr, { tz: timezone });
    return Array.from({ length: count }, () => interval.next().toDate());
  }

  async validateCron(cronExpr: string): Promise<{ valid: boolean; error?: string }> {
    try {
      new CronExpression(cronExpr);
      return { valid: true };
    } catch (e) {
      return { valid: false, error: e.message };
    }
  }

  // Main scheduler loop (runs every 60 seconds)
  async processDueSchedules() {
    const now = new Date();
    const schedules = await db
      .select('*')
      .from('automation_schedules')
      .where('enabled', true);

    for (const schedule of schedules) {
      const isDue = await this.evaluateCron(
        schedule.cron_expression,
        schedule.timezone,
        now
      );

      if (isDue) {
        try {
          // Execute automation
          const execution = await executionService.executeAutomation(
            schedule.automation_id,
            { trigger: 'scheduled' }
          );

          // Record timing
          const drift = Math.abs(
            execution.started_at.getTime() - now.getTime()
          ) / 1000;  // seconds

          await db('automation_executions')
            .where('id', execution.id)
            .update({ scheduled_time: now, execution_drift_seconds: drift });

          // Update next execution time
          const nextExecutions = await this.getNextExecutions(
            schedule.cron_expression,
            schedule.timezone,
            1
          );

          await db('automation_schedules')
            .where('id', schedule.id)
            .update({
              last_execution_at: now,
              next_execution_at: nextExecutions[0]
            });
        } catch (error) {
          logger.error(`Failed to execute scheduled automation ${schedule.automation_id}`, error);
          // Don't throw - continue processing other schedules
        }
      }
    }
  }
}
```

## Dev Notes

- Scheduler is **independent** from execution engine (Story 3.1)
- Cron expressions can be complex; use battle-tested library (cron-parser)
- Timezone handling is **critical** (users expect schedules in their local time)
- Skipped executions are **logged but not retried** (no catch-up behavior)
- Drift tracking helps diagnose reliability issues
- Next execution time is **precomputed** (not calculated on every request)
