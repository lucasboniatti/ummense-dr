## Story

As an **operations user**, I want to **acessar um painel consolidado com navegação lateral, topo global, tarefas e visão temporal**, so that **eu consiga operar o dia de trabalho em uma única superfície de produto**.

---

## Acceptance Criteria

- [ ] Estrutura global com sidebar fixa (Painel, Fluxos, Contatos, Arquivos, Mais) e topbar funcional (busca, filtro, notificações, ações rápidas).
- [ ] Painel principal renderiza lista de tarefas com progresso, prioridade, responsável, tags, vínculo de card e prazo.
- [ ] Painel exibe bloco temporal lateral com calendário mensal e resumo de eventos.
- [ ] A experiência é responsiva (desktop/tablet/mobile) mantendo usabilidade operacional.
- [ ] Estado de loading/erro/vazio é tratado de forma explícita para todos os widgets do painel.
- [ ] Navegação não quebra sessões existentes de webhooks/automations já entregues.

---

## Tasks / Subtasks

- [ ] Definir blueprint UX do shell global (AC: 1, 4, 5).
- [ ] Implementar layout base e navegação global no frontend (AC: 1, 4).
- [ ] Criar widgets de painel para tarefas e calendário/eventos com integração de dados (AC: 2, 3, 5).
- [ ] Padronizar estados de feedback (loading/erro/empty/success) (AC: 5).
- [ ] Validar regressão nas rotas já existentes (`/dashboard/webhooks/*`, `/dashboard/automations/*`) (AC: 6).

---

## Testing

- [ ] Teste de navegação: usuário consegue ir de painel -> fluxos -> card/tarefa sem perda de contexto.
- [ ] Teste de responsividade: 375px, 768px e >=1280px.
- [ ] Teste de regressão: rotas entregues nos Epics 4/5 continuam estáveis.
- [ ] Teste visual: estrutura do shell mantém consistência entre páginas.
