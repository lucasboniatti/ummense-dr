## Dev Notes

### Relevant Context

- O frontend ja possui um primitive `Skeleton` e variantes pré-montadas.
- O valor desta story está em consolidar API, rollout e substituição de loaders manuais, não em recriar o componente.

### API Design

```tsx
import { Skeleton, SkeletonCard, SkeletonTable } from '@/components/ui/Skeleton';

// Basic skeleton
<Skeleton variant="text" width="100%" height={20} />
<Skeleton variant="circular" width={40} height={40} />
<Skeleton variant="rectangular" width={200} height={100} />

// Pre-built skeletons
<SkeletonCard />
<SkeletonTable rows={5} columns={4} />
<SkeletonList items={3} />
```

### CSS Animation

```css
@keyframes skeleton-pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.5; }
}

.skeleton {
  animation: skeleton-pulse 2s ease-in-out infinite;
  background: var(--surface-muted);
}
```

### Relevant Source Tree

- `packages/frontend/src/components/ui/Skeleton.tsx`
- `packages/frontend/src/components/ui/PageLoader.tsx`
- `packages/frontend/src/components/panel/TasksPanel.tsx`
- `packages/frontend/src/components/ExecutionHistoryTable.tsx`
- `packages/frontend/src/components/DLQTable.tsx`

### Implementation Guidance

- Reutilizar o primitive existente e reduzir a quantidade de loading states manuais.
- Validar consistência visual com `PageLoader` e `Spinner`.
- Evitar introduzir nova taxonomia de variants se os helpers atuais já cobrirem a maior parte dos casos.

### Evidence Expectations

- Lista dos loaders manuais substituídos.
- Before/after de pelo menos uma tabela/lista com skeleton compartilhado.
