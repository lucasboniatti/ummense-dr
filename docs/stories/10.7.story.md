## Story

As a **user**, I want to **navegar entre diferentes seções de conteúdo com abas (Tabs)**, so that **eu possa organizar e acessar informações de forma estruturada**.

---

## Acceptance Criteria

- [ ] Componente Tabs implementado com Radix UI Tabs.
- [ ] TabsList com indicador visual de aba ativa.
- [ ] TabsTrigger com estados: idle, active, hover, focus.
- [ ] TabsContent renderiza conteúdo da aba selecionada.
- [ ] Navegação por teclado (Arrow keys, Home, End).
- [ ] Acessível com role="tablist", role="tab", role="tabpanel".
- [ ] Dark mode suportado.
- [ ] Animação de transição entre abas.

## Dev Notes

### Relevant Context

- O frontend atual ainda não tem um primitive compartilhado de tabs, mas possui superfícies candidatas com conteúdo seccionado.
- O objetivo principal desta story é criar o primitive reutilizável sem expandir escopo funcional.

### API Design

```tsx
import { Tabs, TabsList, TabsTrigger, TabsContent } from '@/components/ui/Tabs';

<Tabs defaultValue="overview">
  <TabsList>
    <TabsTrigger value="overview">Visão Geral</TabsTrigger>
    <TabsTrigger value="history">Histórico</TabsTrigger>
    <TabsTrigger value="settings">Configurações</TabsTrigger>
  </TabsList>
  <TabsContent value="overview">Conteúdo da visão geral</TabsContent>
  <TabsContent value="history">Conteúdo do histórico</TabsContent>
  <TabsContent value="settings">Conteúdo das configurações</TabsContent>
</Tabs>
```

### Variants

```tsx
// Underline style (default)
<Tabs variant="underline">

// Pills style
<Tabs variant="pills">
```

### Relevant Source Tree

- `packages/frontend/src/components/ui/Tabs.tsx`
- `packages/frontend/src/components/ExecutionDetailModal.tsx`
- `packages/frontend/src/pages/dashboard/integrations/index.tsx`
- `packages/frontend/src/pages/cards/[cardId].tsx`

### Implementation Guidance

- Validar acessibilidade, foco e teclado antes de pensar em múltiplos variants visuais.
- Não deixar o primitive completamente órfão; registrar a primeira surface candidata mesmo que a adoção final fique para uma story subsequente.
- Não criar backend nem estado global novo para suportar tabs.

### Evidence Expectations

- Demonstração do primitive em uso real ou prova clara da superfície candidata escolhida.
- Evidência de keyboard navigation e dark mode.
