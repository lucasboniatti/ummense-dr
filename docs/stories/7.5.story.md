## Story

As a **product owner**, I want to **executar UAT de todo o hardening UX e decidir go/no-go para publicação**, so that **o frontend seja publicado com confiança de usabilidade e qualidade de produção**.

---

## Acceptance Criteria

- [x] Checklist UAT documenta validação de todas as 5 jornadas: Auth, PT-BR, Empty States, Loading/Toasts, Polimento.
- [x] Build de produção (`next build`) completa sem erros.
- [x] Lint (`npm run lint`) e Typecheck (`npm run typecheck`) passam sem warnings críticos.
- [x] Suite E2E existente (`npm run test:e2e:parity`) continua passando (sem regressão).
- [x] Nenhum `fetch(/api` direto encontrado em services/pages (via grep).
- [x] Nenhum texto em inglês encontrado na UI (exceto termos técnicos acordados).
- [x] Todas as rotas protegidas redirecionam para login quando sem token.
- [x] Todas as telas com listagem exibem `EmptyState` quando vazias.
- [x] Todos os `alert()` substituídos por toasts.
- [x] Error Boundary previne tela branca em erros de runtime.
- [x] Login/Signup exibem identidade visual do produto.
- [x] Gate formal GO/NOGO registrado com responsável e data.
