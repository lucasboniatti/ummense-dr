## Story

As an **operations user**, I want to **ter minha sessão protegida com login obrigatório e poder sair do sistema**, so that **apenas usuários autenticados acessem o painel operacional e eu tenha controle da minha sessão**.

---

## Acceptance Criteria

- [x] Rotas protegidas redirecionam automaticamente para `/auth/login` quando não há token válido.
- [x] Rotas `/auth/*` (login, signup, integration-callback) NÃO são protegidas pelo guard e funcionam sem token.
- [x] `AuthContext` expõe `user`, `token`, `isAuthenticated`, `isLoading`, `login()`, `signup()`, `logout()`.
- [x] `AuthContext` valida token existente ao montar via `GET /api/auth/me`; se inválido, limpa sessão e redireciona.
- [x] Tela de loading com spinner aparece enquanto sessão está sendo validada (entre abrir app e render da página).
- [x] Sidebar exibe nome/email do usuário logado no footer e botão "Sair" funcional.
- [x] Logout limpa `localStorage` (chaves `synkra_dev_token` e `token`), chama `POST /api/auth/logout` e redireciona para `/auth/login`.
- [x] Token é mantido na chave `synkra_dev_token` (retrocompatibilidade com sistema atual).

## API Contract (Backend já implementado)

```
POST /api/auth/login     → { email, password } → { token, user: { id, email } }
POST /api/auth/signup    → { email, password } → { token, user: { id, email } }
GET  /api/auth/me        → (Bearer token) → { id, email }
POST /api/auth/logout    → { success: true }
```

## Technical Context for Dev Agent

### Arquivo a criar: `src/contexts/AuthContext.tsx`

```typescript
interface AuthUser {
  id: string;
  email: string;
  name?: string;
}

interface AuthContextType {
  user: AuthUser | null;
  token: string | null;
  isLoading: boolean;
  isAuthenticated: boolean;
  login: (email: string, password: string) => Promise<void>;
  signup: (email: string, password: string) => Promise<void>;
  logout: () => void;
}
```

**Regras:**
1. Token armazenado em `localStorage` com chave `synkra_dev_token`.
2. Ao montar, chamar `GET /api/auth/me` para validar token existente.
3. Se inválido/expirado → limpar `localStorage` e redirecionar para `/auth/login`.
4. Expor via hook `useAuth()`.

### Arquivo a criar: `src/components/guards/AuthGuard.tsx`

**Regras:**
1. Envolver o `AppShell` (não cada página individual).
2. Enquanto `isLoading === true`, exibir tela de loading com spinner centralizado.
3. Se `!isAuthenticated`, redirect para `/auth/login` via `router.replace()`.
4. Se authenticated, renderizar children normalmente.

### Modificação em `src/pages/_app.tsx`

```tsx
// Estrutura resultante:
<AuthProvider>
  {isAuthRoute ? (
    <Component {...pageProps} />
  ) : (
    <AuthGuard>
      <AppShell>
        <Component {...pageProps} />
      </AppShell>
    </AuthGuard>
  )}
</AuthProvider>
```
