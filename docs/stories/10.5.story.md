## Story

As a **user**, I want to **marcar e desmarcar opções com um componente Checkbox acessível**, so that **eu possa fazer seleções múltiplas de forma clara e intuitiva**.

---

## Acceptance Criteria

- [ ] Componente Checkbox implementado com Radix UI Checkbox.
- [ ] Estados: unchecked, checked, indeterminate.
- [ ] Animação suave ao marcar/desmarcar.
- [ ] Ícone de check visível quando marcado.
- [ ] Label associado ao checkbox.
- [ ] Navegação por teclado (Space para toggle).
- [ ] Acessível com aria-checked.
- [ ] Dark mode suportado.
- [ ] Integrado com react-hook-form.
- [ ] Aplicado em pelo menos uma surface atual com checkbox nativo.

## Dev Notes

### Relevant Context

- O frontend atual usa checkboxes nativos em formulários e tabelas operacionais.
- Esta story deve criar o primitive e substitui-lo em pelo menos um caso real para evitar componente órfão.

### API Design

```tsx
import { Checkbox } from '@/components/ui/Checkbox';

<Checkbox
  checked={isChecked}
  onCheckedChange={setIsChecked}
  label="Aceito os termos"
/>

// Com indeterminate
<Checkbox
  checked="indeterminate"
  label="Selecionar todos"
/>
```

### Relevant Source Tree

- `packages/frontend/src/components/WebhookForm.tsx`
- `packages/frontend/src/components/TaskList.tsx`
- `packages/frontend/src/components/DLQTable.tsx`
- `packages/frontend/src/components/ScheduleToggleControl.tsx`
- `packages/frontend/src/components/ui/Checkbox.tsx`

### Implementation Guidance

- Manter suporte a uso controlado e não controlado conforme o contrato do Radix.
- Garantir label associado e área de clique confortável para uso operacional.
- Evitar expandir escopo para regras de formulário; foco aqui é primitive + adoção inicial.

### Evidence Expectations

- Before/after da surface real escolhida.
- Evidência de keyboard toggle, indeterminate e dark mode.
