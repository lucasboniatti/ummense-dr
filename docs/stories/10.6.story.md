## Dev Notes

### Relevant Context

- `TriggerTypeSelector.tsx` já usa radios nativos para alternar entre `webhook` e `scheduled`.
- Esta story deve priorizar esse uso real em vez de criar apenas um primitive isolado.

### API Design

```tsx
import { RadioGroup, RadioItem } from '@/components/ui/RadioGroup';

<RadioGroup value={value} onValueChange={setValue}>
  <RadioItem value="low" label="Baixa prioridade" />
  <RadioItem value="medium" label="Média prioridade" />
  <RadioItem value="high" label="Alta prioridade" />
</RadioGroup>
```

### Relevant Source Tree

- `packages/frontend/src/components/TriggerTypeSelector.tsx`
- `packages/frontend/src/components/ui/RadioGroup.tsx`
- `packages/frontend/src/components/TaskForm.tsx` (se algum fluxo futuro precisar de escolha exclusiva)

### Implementation Guidance

- Manter navegação por teclado correta com setas e foco visível.
- Preservar semântica de grupo/itens e labels clicáveis.
- Não expandir a story para backend ou lógica de negócio do trigger.

### Evidence Expectations

- Before/after de `TriggerTypeSelector.tsx`.
- Evidência de teclado, disabled state e dark mode.
