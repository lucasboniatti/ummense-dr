## Story

As a **power user**, I want to **create multi-step workflows with conditional branching (if/then/else)**, so that **I can build complex automation scenarios like approval chains and multi-stage pipelines**.

## Acceptance Criteria

- [ ] **[CRITICAL]** Workflow builder supports DAG (Directed Acyclic Graph) visualization
  - Drag-and-drop canvas to add steps
  - Visual lines connecting steps (arrows showing flow direction)
  - Node styling: Start (green), Step (blue), Conditional (yellow), End (red)
  - Zoom and pan for large workflows
  - Grid snap for alignment
  - No cycles allowed (validation prevents loops)
- [ ] **[CRITICAL]** Workflow steps support: Sequential, Parallel, Conditional (if/then/else), Loop
  - Sequential: One step after another
  - Parallel: Multiple steps run simultaneously
  - Conditional: if/then/else based on step output or context
  - Loop: Repeat step until condition met
  - Each step type has different visual representation in builder
- [ ] **[HIGH]** Conditions can reference previous step outputs, automation context, and external API responses
  - Condition editor supports expressions: `step1.result === "approved"`, `context.user_role === "admin"`
  - Syntax highlighting and autocomplete for available variables
  - Expression evaluation at runtime uses safe JavaScript evaluator (no `eval()`)
  - Supports operators: ==, !=, <, >, <=, >=, &&, ||, !
- [ ] **[HIGH]** Step branches are visually distinct in builder (green for if, red for else)
  - Successful condition path: Green line/label "If True"
  - Failed condition path: Red line/label "If False"
  - Color coding helps users understand flow at a glance
- [ ] **[HIGH]** Execution engine correctly processes branched workflows (only active branches execute)
  - At runtime, conditional evaluation determines which branch executes
  - Inactive branches are skipped (not executed, not logged)
  - Branch results properly merged for downstream steps
  - Parallel branches wait for all to complete before continuing
- [ ] **[MEDIUM]** Complex workflows can be saved as templates for reuse
  - Save button: "Save as Template" (modal to name template)
  - Template library: View, search, load existing templates
  - Template sharing: Can be shared with team members (permission model)
  - Version templates (e.g., "Approval Chain v1", "Approval Chain v2")
- [ ] **[MEDIUM]** Test workflow execution (dry-run) with sample data before publishing
  - Dry-run button: Execute workflow with sample data without triggering external actions
  - Sample data editor: Provide test inputs for first step
  - Dry-run results: Show execution path, outputs, final result
  - No webhooks sent, no external side effects during dry-run
- [ ] **[MEDIUM]** Existing single-step and simple workflows upgrade gracefully to new builder
  - Automatically convert Wave 2 automations to new workflow format
  - One-way conversion (no downgrade back to Wave 2 format)
  - Auto-migration happens silently on first edit
  - User sees workflow as single-step in new builder

## Technical Notes

### DAG Workflow Model

```typescript
interface WorkflowStep {
  id: string;                           // Unique step ID
  type: 'webhook' | 'conditional' | 'action' | 'loop';
  name: string;                         // Display name
  config: Record<string, unknown>;      // Step-specific config
  position?: { x: number; y: number };  // For canvas rendering
}

interface Conditional {
  condition: string;                    // Expression: "step1.result === 'approved'"
  ifTrue: string[];                     // Step IDs to execute if true
  ifFalse: string[];                    // Step IDs to execute if false
}

interface Workflow {
  id: string;
  userId: string;
  name: string;
  version: number;
  steps: Map<string, WorkflowStep>;     // stepId -> step definition
  edges: Array<{                        // DAG edges
    from: string;
    to: string;
    type?: 'sequential' | 'conditional' | 'parallel';
    condition?: Conditional;
  }>;
  metadata: {
    createdAt: Date;
    updatedAt: Date;
    templateId?: string;                // If saved as template
  };
}

interface WorkflowTemplate {
  id: string;
  name: string;
  version: number;
  workflow: Workflow;
  createdBy: string;
  sharedWith: string[];                 // User IDs
  description?: string;
}
```

### Conditional Logic Evaluator

```typescript
// Safe expression evaluator (NO eval(), NO dangerous functions)
class ExpressionEvaluator {
  private allowedOperators = ['==', '!=', '<', '>', '<=', '>=', '&&', '||', '!'];
  private forbiddenPatterns = ['eval', 'Function', 'setTimeout', 'fetch', 'import'];

  async evaluate(
    expression: string,
    context: Record<string, unknown>
  ): Promise<boolean> {
    // Sanitize: block forbidden patterns
    for (const pattern of this.forbiddenPatterns) {
      if (expression.includes(pattern)) {
        throw new Error(`Forbidden pattern in expression: ${pattern}`);
      }
    }

    // Use safe evaluator (jsonata, jmespath, or custom parser)
    // Example: jsonata library
    const result = jsonata(expression).evaluate(context);
    return Boolean(result);
  }

  // Validate expression syntax
  validateExpression(expression: string): { valid: boolean; error?: string } {
    try {
      jsonata(expression);
      return { valid: true };
    } catch (e) {
      return { valid: false, error: e.message };
    }
  }
}
```

### Workflow Execution with Branching

```typescript
async function executeWorkflow(workflow: Workflow, context: ExecutionContext) {
  const executedSteps = new Map<string, unknown>();
  const stepOrder = topologicalSort(workflow.edges);  // DAG ordering

  for (const stepId of stepOrder) {
    const step = workflow.steps.get(stepId);
    if (!step) continue;

    // Check if this step should execute (parent conditions)
    const shouldExecute = await checkParentConditions(stepId, workflow, executedSteps);
    if (!shouldExecute) continue;

    // Execute step
    const result = await executeStep(step, context, executedSteps);
    executedSteps.set(stepId, result);

    // Check for conditional branches
    if (step.type === 'conditional') {
      const condition = step.config.condition as Conditional;
      const conditionMet = evaluator.evaluate(condition.condition, {
        ...context,
        stepOutputs: Object.fromEntries(executedSteps)
      });

      if (conditionMet) {
        // Mark "if true" branch for execution
        for (const nextStepId of condition.ifTrue) {
          markForExecution(nextStepId);
        }
      } else {
        // Mark "if false" branch for execution
        for (const nextStepId of condition.ifFalse) {
          markForExecution(nextStepId);
        }
      }
    }
  }

  return executedSteps;
}
```
