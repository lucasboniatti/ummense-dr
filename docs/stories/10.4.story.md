## Story

As a **user**, I want to **selecionar opções de dropdowns com um componente Select acessível e estilizado**, so that **eu possa fazer escolhas de forma intuitiva e com boa experiência visual**.

---

## Acceptance Criteria

- [ ] Componente Select implementado com Radix UI Select primitive.
- [ ] Suporta variants: default, error, disabled.
- [ ] Suporta sizes: sm, md, lg.
- [ ] Placeholder exibido quando nenhuma opção selecionada.
- [ ] Ícone de chevron indica que é dropdown.
- [ ] Lista de opções com scroll quando muitas opções.
- [ ] Navegação por teclado (Arrow keys, Enter, Escape).
- [ ] Acessível com aria-label, aria-expanded, aria-selected.
- [ ] Dark mode suportado.
- [ ] Integrado com react-hook-form via Controller.
- [ ] Aplicado em pelo menos uma surface atual que hoje usa `<select>` nativo.

## Dev Notes

### Relevant Context

- O frontend atual ainda usa `<select>` nativo em múltiplas superfícies, incluindo formulários e filtros.
- O componente novo precisa funcionar com `react-hook-form`, mas sem obrigar a migração completa dos formulários na mesma story.
- O primeiro rollout deve acontecer em uma superfície real do produto, não apenas em demo isolada.

### API Design

```tsx
import { Select, SelectTrigger, SelectContent, SelectItem } from '@/components/ui/Select';

<Select value={value} onValueChange={setValue}>
  <SelectTrigger placeholder="Selecione uma opção" />
  <SelectContent>
    <SelectItem value="low">Baixa</SelectItem>
    <SelectItem value="medium">Média</SelectItem>
    <SelectItem value="high">Alta</SelectItem>
  </SelectContent>
</Select>
```

### Integração com React Hook Form

```tsx
<Controller
  name="priority"
  control={control}
  render={({ field }) => (
    <Select value={field.value} onValueChange={field.onChange}>
      ...
    </Select>
  )}
/>
```

### Relevant Source Tree

- `packages/frontend/src/components/TaskForm.tsx`
- `packages/frontend/src/components/TriggerTypeSelector.tsx`
- `packages/frontend/src/components/layout/AppTopbar.tsx`
- `packages/frontend/src/components/ExecutionHistoryTable.tsx`
- `packages/frontend/src/components/ui/Select.tsx`

### Implementation Guidance

- Priorizar um contrato de API simples e compatível com `Controller`.
- Preservar teclado, foco e comportamento de scroll para listas longas.
- Evitar qualquer dependência de backend; a story e estritamente de UI e formulario.

### Evidence Expectations

- Before/after do primeiro formulário migrado.
- Evidência de teclado e dark mode no primitive.
