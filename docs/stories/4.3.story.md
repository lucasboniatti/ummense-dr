### Current app.ts State
[Source: packages/backend/src/app.ts]
- Currently mounts: `/api/webhooks`, `/api/analytics`
- TaskFlow routes (flows, cards, tasks) are NOT mounted — they exist as orphaned files
- Auth routes exist at `/routes/auth.ts` but may also be unmounted

### Auth Middleware Import
[Source: packages/backend/src/middleware/auth.middleware.ts]
```typescript
import { authMiddleware, AuthRequest } from '../middleware/auth.middleware';
```
**WARNING:** Some existing route files import from `../middleware/auth` (non-existent). Use `auth.middleware` consistently.

### Auth ID Type Mismatch
[Source: docs/architecture/taskflow-architecture.md#6.1]
- Use service role Supabase client and filter by user_id manually (same pattern as `webhook.service.ts`)
- Do NOT try to map types — treat user_id as the value from `req.user.id`

### types/user.ts Issue
[Source: git status]
- File shows as deleted (` D packages/backend/src/types/user.ts`)
- `auth.controller.ts` imports `User` and `AuthPayload` from it
- Contents were: `User { id: number; email: string; password_hash: string; created_at: Date; updated_at: Date }` and `AuthPayload { email: string; password: string }`

### Route Path Reference
[Source: docs/architecture/taskflow-architecture.md#3.2-3.4]
```
GET/POST      /api/flows
GET/PUT/DELETE /api/flows/:flowId
POST          /api/flows/:flowId/columns
PUT/DELETE    /api/flows/:flowId/columns/:columnId
PUT           /api/flows/:flowId/columns/reorder

GET/POST      /api/cards
GET/PUT/DELETE /api/cards/:cardId
PUT           /api/cards/reorder

GET/POST      /api/tasks
GET/PUT/DELETE /api/tasks/:taskId
```
