## Story

As a **user with mission-critical webhooks**, I want to **receive automatic retry with exponential backoff when webhook delivery fails**, so that **temporary network issues don't cause permanent data loss**.

---

## Acceptance Criteria

- [ ] **[CRITICAL]** Failed webhook deliveries retry with exponential backoff delays: 1s, 5s, 30s, 5m, 30m
  - Retry attempt 1: 1 second after initial failure
  - Retry attempt 2: 5 seconds after first retry
  - Retry attempt 3: 30 seconds after second retry
  - Retry attempt 4: 5 minutes after third retry
  - Retry attempt 5: 30 minutes after fourth retry
  - Total time window: ~36 minutes for all retries
  - Failures: HTTP 4xx/5xx, connection timeouts, DNS failures
- [ ] **[CRITICAL]** After final retry, webhook moved to Dead Letter Queue (DLQ) for manual review
  - DLQ table stores: webhook_id, automation_id, delivery_attempt_count, last_error, timestamp
  - DLQ records immutable (no auto-deletion)
  - Manual retry capability: UI button to retry failed webhooks
  - Batch retry: Select multiple DLQ items and retry simultaneously
- [ ] **[CRITICAL]** DLQ interface shows failed webhooks with full error context and retry capability
  - DLQ dashboard tab: List of failed webhooks (paginated)
  - Columns: Webhook URL, Destination, Last failure time, Error message, Automation name
  - Expandable detail: Full error context, last payload, all retry attempts
  - Manual retry button: Resend immediately
  - Clear button: Remove from DLQ (after review)
- [ ] **[HIGH]** Manual retry from DLQ works and re-attempts delivery
  - Clicking "Retry" on DLQ item resets retry_count and starts from attempt 1
  - Manual retry follows same exponential backoff schedule
  - Success removes from DLQ automatically
  - Failure increments retry_count (max 5 again) and returns to DLQ
- [ ] **[HIGH]** Webhook signature validation (HMAC-SHA256) implemented
  - Every webhook delivery includes `X-Synkra-Signature` header: SHA256(payload + secret)
  - Signature computed using automation's webhook_secret from database
  - Destination systems must validate signature to confirm authenticity
  - Signature validation prevents tampering with webhook payloads
  - Documentation: Include signature format and validation example
- [ ] **[HIGH]** Audit log captures all delivery attempts with timestamps and outcomes
  - Table: webhook_deliveries - id, webhook_id, attempt_number, status, error_message, timestamp
  - Captured for each attempt: URL, payload, response status, duration
  - Audit logs immutable (append-only)
  - Queryable via API: "Show delivery history for webhook X"
- [ ] **[HIGH]** Database stores webhook delivery history for minimum 90 days
  - Retention policy configurable (default 90 days, max 2 years)
  - Automated cleanup job: Delete records >90 days old nightly
  - Retention setting stored per user (automation settings)
- [ ] **[MEDIUM]** Performance: Retry service processes 1000+ failed webhooks concurrently
  - Async processing (non-blocking)
  - Queue-based retry scheduling (Bull, node-cron, or similar)
  - P99 latency for retry scheduling <100ms

## Technical Notes

### Webhook Delivery Architecture

```
automation execution finishes
    ↓
FOR EACH webhook in automation.webhooks:
  1. Build webhook payload (execution context + outputs)
  2. Generate HMAC-SHA256 signature
  3. Attempt delivery (HTTP POST with signature header)
  4. If success (2xx): log and continue
  5. If failure (4xx/5xx/timeout): schedule retry
  ↓
Retry service (async process):
  - Run every 1 minute
  - For each pending retry: check delay window
  - If delay elapsed: attempt delivery
  - If success: remove from retry queue, log success
  - If failure: reschedule with next backoff interval
  - If max retries reached: move to DLQ
```

### HMAC-SHA256 Signature Implementation

```typescript
// In webhook delivery service
function generateSignature(payload: object, secret: string): string {
  const payloadStr = JSON.stringify(payload);
  const signature = createHmac('sha256', secret)
    .update(payloadStr)
    .digest('hex');
  return signature;
}

// Header format
function buildWebhookHeaders(signature: string): Record<string, string> {
  return {
    'Content-Type': 'application/json',
    'X-Synkra-Signature': signature,
    'X-Synkra-Timestamp': new Date().toISOString(),
    'User-Agent': 'Synkra/3.0'
  };
}

// Destination system validates (example in docs)
function validateWebhookSignature(body: string, signature: string, secret: string): boolean {
  const expectedSignature = createHmac('sha256', secret)
    .update(body)
    .digest('hex');
  return signature === expectedSignature;  // Constant-time comparison recommended
}
```

### Retry Service (Async Background Job)

```typescript
// Task: Every 60 seconds, process pending retries
async function processWebhookRetries() {
  const pendingRetries = await db
    .select('*')
    .from('webhook_deliveries')
    .where('status', 'pending')
    .where('next_retry_at', '<=', now());

  for (const delivery of pendingRetries) {
    try {
      const response = await fetch(delivery.webhook_id, {
        method: 'POST',
        headers: buildWebhookHeaders(delivery.signature),
        body: JSON.stringify(delivery.payload),
        timeout: 30000
      });

      if (response.ok) {
        // Success
        await updateDelivery(delivery.id, { status: 'success' });
        await deleteDLQItem(delivery.id);  // Remove from DLQ if there
      } else {
        // Retry or DLQ
        if (delivery.attempt_number < 5) {
          const nextInterval = getBackoffInterval(delivery.attempt_number);
          await scheduleRetry(delivery.id, nextInterval);
        } else {
          // Move to DLQ
          await moveToDLQ(delivery.id);
        }
      }
    } catch (error) {
      if (delivery.attempt_number < 5) {
        const nextInterval = getBackoffInterval(delivery.attempt_number);
        await scheduleRetry(delivery.id, nextInterval, error.message);
      } else {
        await moveToDLQ(delivery.id, error.message);
      }
    }
  }
}

function getBackoffInterval(attemptNumber: number): number {
  const intervals = [1, 5, 30, 300, 1800];  // seconds
  return intervals[attemptNumber] * 1000;  // Convert to ms
}
```
