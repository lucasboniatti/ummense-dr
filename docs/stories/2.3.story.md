## Technical Notes

### SSRF Validation Rules
```javascript
const ssrfBlocklist = [
  '127.0.0.0/8',      // localhost
  '10.0.0.0/8',       // private
  '172.16.0.0/12',    // private
  '192.168.0.0/16',   // private
  '::1',              // IPv6 localhost
  'fc00::/7',         // IPv6 private
];

// Validation logic:
1. Parse URL, extract hostname
2. Resolve hostname to IP
3. Check if IP in blocklist
4. Block if yes, proceed if no
```

### Retry Strategy
```
Attempt 1: Immediate
Attempt 2: 2^1 = 2 seconds
Attempt 3: 2^2 = 4 seconds
Attempt 4: 2^3 = 8 seconds
Attempt 5: 2^4 = 16 seconds
Total time: ~30 seconds across 5 attempts
Max single backoff: 300 seconds (5 minutes)
```

### Idempotency Implementation
```javascript
const idempotencyKey = sha256(`${webhookId}:${eventId}:${eventType}`);
// Header sent: X-Idempotency-Key: {idempotencyKey}
// External system should use this to deduplicate
// If receives same key, should return same response
```

### Dead Letter Queue Pattern
- After 5 failed attempts → Move to dead-lettered status
- Record final error message in response_body
- Alert monitoring system
- Manual review required (create ticket in ops system)
- Allow manual retry via admin API

## Dev Notes

- Webhook delivery is asynchronous (fire-and-forget at event time, background retry job handles retries)
- SSRF validation performed at webhook creation + before each delivery attempt
- Idempotency key prevents duplicate processing on external system
- 30-second timeout covers most cloud APIs (AWS, Supabase, Stripe)
- Retry job runs synchronously (scheduled task), not queue-based
