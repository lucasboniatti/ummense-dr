## Story

As a **developer**, I want to **os design tokens em tokens.yaml serem a fonte da verdade para CSS e Tailwind**, so that **todas as cores, espaçamentos e tipografias sejam consistentes e sincronizadas**.

---

## Acceptance Criteria

- [x] `src/tokens/tokens.yaml` permanece como a fonte da verdade dos tokens usados pelo frontend.
- [x] Drift entre `src/tokens/index.ts`, `packages/frontend/src/styles/globals.css` e `packages/frontend/tailwind.config.js` é auditado e reduzido.
- [x] O frontend consome tokens alinhados de cor, spacing, typography, radius e shadow sem quebrar as rotas já validadas.
- [x] Se uma automação de geração for introduzida, a migração é documentada e não rompe os consumidores atuais.
- [x] Dark mode tokens aplicados corretamente via `[data-theme="dark"]`.
- [x] TypeScript exports tipos seguros para tokens.

## Dev Notes

### Relevant Context

- O repositório ja possui `src/tokens/tokens.yaml` e `src/tokens/index.ts`; o problema atual e alinhamento e governança, nao ausencia completa de token source.
- O frontend ainda carrega valores duplicados em `globals.css` e `packages/frontend/tailwind.config.js`.
- A story nao deve assumir adoção de tooling novo sem documentar a migração no mesmo fluxo.

### Estrutura Atual

```
src/tokens/
├── tokens.yaml      # Source of truth
└── index.ts         # exports atuais
```

### Relevant Source Tree

- `src/tokens/tokens.yaml`
- `src/tokens/index.ts`
- `packages/frontend/src/styles/globals.css`
- `packages/frontend/tailwind.config.js`
- `tailwind.config.ts`
- `package.json`

### Implementation Guidance

- Priorizar alinhamento incremental sobre reescrita total da pipeline.
- Se `style-dictionary` ou automação similar for adotado na mesma story, a migração deve substituir a duplicação existente sem deixar duas fontes de verdade.
- Se a automação for deferida, registrar isso na documentação da story e ainda reduzir o drift entre tokens e frontend.
- Validar as rotas `/dashboard`, `/dashboard/automations`, `/dashboard/webhooks`, `/cards/[cardId]` e `/auth/login` após qualquer ajuste de token.

### Evidence Expectations

- Relato curto dos pontos de drift corrigidos.
- Before/after das superfícies mais afetadas.
- Evidência de `npm run build` após o alinhamento.
