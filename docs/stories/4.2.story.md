### Auth ID Type Mismatch
[Source: docs/architecture/taskflow-architecture.md#6.1]
- Current auth uses `id: number` in JWT, DB schema uses UUID
- **Required bridge for this story:** Resolve the Supabase auth user UUID from `req.user.email` using the service-role client (`auth.users.email -> auth.users.id`) before any `flows.user_id`, `cards.created_by`, `tasks.created_by`, or ownership-filtered query
- Do NOT filter UUID columns using the numeric `req.user.id`
- If the authenticated email cannot be resolved to an `auth.users` row, return a consistent error response and stop the request

### Default Columns for createFlow
[Source: docs/architecture/taskflow-architecture.md#3.2]
```
Backlog (#6B7280), A Fazer (#3B82F6), Em Progresso (#F59E0B), Finalizado (#10B981)
```

### Error Response Format
[Source: docs/architecture/taskflow-architecture.md#3.5]
```typescript
{ error: "message", code: "VALIDATION_ERROR" | "NOT_FOUND" | "FORBIDDEN" | "CONFLICT" | "INTERNAL_ERROR" }
// 400, 401, 403, 404, 409, 500
```
